title: 配置概述
description: 了解配置 Labelgate 的不同方式。
---

Labelgate 支持三种配置模式。所有选项遵循统一的命名规范 — 完整列表请查看[配置参考](/zh/docs/configuration/reference)。

## 配置模式

### 1. 仅环境变量（推荐用于简单设置）

无需配置文件。使用 `LABELGATE_` 前缀设置环境变量：

```bash
LABELGATE_CLOUDFLARE_API_TOKEN=your-token
LABELGATE_CLOUDFLARE_ACCOUNT_ID=your-account-id
LABELGATE_CLOUDFLARE_TUNNEL_ID=your-tunnel-id
```

这是最简单的方法，在[快速开始](/zh/docs/getting-started)指南中使用。

### 2. 配置文件

使用 YAML 配置文件进行更复杂的设置（例如，多凭证、多隧道）。

```yaml
# labelgate.yaml
cloudflare:
  api_token: your-cloudflare-api-token
  account_id: your-account-id
  tunnel_id: your-tunnel-id
```

通过 `LABELGATE_CONFIG` 环境变量指定配置文件路径：

```bash
LABELGATE_CONFIG=/path/to/labelgate.yaml
```

如果未设置 `LABELGATE_CONFIG`，Labelgate 也会自动查找 `./labelgate.yaml` 和 `/etc/labelgate/labelgate.yaml`。

### 3. 混合模式

结合两者：使用配置文件存放非敏感设置，通过环境变量传入密钥。

**优先级：环境变量 > 配置文件 > 默认值**

```yaml
# labelgate.yaml - 非敏感配置
cloudflare:
  account_id: your-account-id
  tunnel_id: your-tunnel-id
```

```bash
# 密钥通过环境变量传入（会覆盖配置文件中的对应值）
LABELGATE_CLOUDFLARE_API_TOKEN=your-token
```
